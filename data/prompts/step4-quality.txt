直前のステップで出力した記事を人間編集者の視点で品質向上させ、修正済みの記事を出力してください。

【編集原則】
1. 読者ファースト：読者を否定せず、求める情報を最短距離で提供する
2. 断定と責任：曖昧な表現を排除し、責任を持った断定を行う
3. 具体性の追求：抽象的な表現を具体的な数値・条件・行動に置換する
4. AI臭の排除：機械的・テンプレート的な表現を人間らしい文章に変換する
5. 可読性の最大化：1文60文字以内、漢字:ひらがな=3:7

【絶対禁止表現リスト（検出したら修正必須）】

▼A. 読者否定型比較表現
- 〜と考えている方が多いのですが → 結論先行形式に変換
- 〜と思い込んでいる方が多いのですが → 結論先行形式に変換
- 多くの人は〜と思えますが → 削除して結論から開始
- 〜から始めがちですが → 削除して推奨行動から開始
- 〜と思われがちですが → 削除して結論から開始

▼B. 主語不明瞭表現
- こんな声も聞かれます → 具体的人物像＋引用形式に変換
- 〜という声も上がっています → 報告元を明記
- 〜という方もいます → 具体的人物像を主語に

▼C. AI典型問いかけパターン
- 〜ではないでしょうか？ → 断定形に変換
- 〜と思いませんか？ → 断定形に変換
- 〜ですよね？ → 断定形に変換
- 〜で悩んでいませんか？ → 削除して解決策を提示
- 〜をご存知でしょうか？ → 削除して事実を述べる

▼D. AI典型前置きパターン
- 実は〜なんです → 「実は」を削除して断定
- ここで重要なのは〜 → 削除して本題を直接述べる
- 意外と知られていないのですが〜 → 削除して事実を述べる
- 結論から言うと〜 → 「結論から言うと」を削除して結論を述べる

▼E. 責任回避・専門家丸投げ
- 税理士にご相談ください → 削除
- 専門家にご相談ください → 削除
- ケースバイケースです → 削除
- 状況によって異なります → 削除
- 同様の結果を保証するものではありません → 削除

▼F. 曖昧・責任回避表現
- 〜の可能性があります → 条件を明示して断定
- 〜かもしれません → 条件を明示して断定
- 〜と言われています → 出典を明示、または断定
- 〜とされています → 出典を明示、または断定
- 基本的には〜 → 「基本的には」を削除して断定
- 一般的には〜 → 「一般的には」を削除して断定
- 多くの場合〜 → 具体的な割合・条件を明示

▼G. 冗長表現
- 〜することができます → 〜できます
- 〜する必要があります → 〜してください
- 〜することが重要です → 〜してください
- 〜することをおすすめします → 〜してください
- 〜について解説します → 削除して本題へ
- 〜について見ていきましょう → 削除して本題へ
- 〜になってきます → 〜になります

▼H. 抽象的評価語（理由・効果を具体的に述べる）
- 重要です / 大切です / 必要です / 有効です / 効果的です / 便利です
- 注意が必要です / 確認が必要です / 検討が必要です

▼I. 弱い指示表現（「〜してください」に統一）
- 〜しましょう / 〜するといいでしょう / 〜がおすすめです
- 〜してみましょう / 〜を推奨します

▼Q. AI特有の表現パターン
- 「〜という点が挙げられます」 → 「〜があります」
- 「それでは〜」 → 削除（前置き不要）
- 「包括的な」「革新的な」「効率的な」 → 「幅広い」「新しい」「便利な」
- 「一般的に」「多くの人が」 → 「よく」「みんな」「7割の人が」
- 「実際にやってみると」 → 削除（AI的な体験偽装表現）

【編集禁止要素】
- C工程で確認済みの数値データ・固有名詞・法律名・日付
- H2/H3見出しの順序と構成
- 出典・参照元セクション全体（1文字も変更しない）

【出力形式】
記事全文を [ARTICLE_DRAFT] タグ（開始）と対応する閉じタグ [/ARTICLE_DRAFT] で囲んで出力すること。
タグ外への出力禁止。meta_description: 行はタグ直後に1行のみ出力する。

【出力禁止事項】
- 「〇件修正しました」等の作業報告
- タグの外にテキストを出力すること
- 採点結果・品質スコア・修正箇所のハイライトや説明
